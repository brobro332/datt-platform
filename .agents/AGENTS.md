# DATT 플랫폼 개발 규칙

## 1. 개발자 노트 자동 업데이트 규정
- 모든 신규 기능 개발, 버그 픽스, 리팩토링 완료 후 커밋 및 푸시 명령을 실행하기 전에 반드시 개발자 노트 문서를 업데이트해야 합니다.
- 개발자 노트는 `docs/DEVELOPMENT_NOTE_v[버전].md` 파일 형식으로 관리됩니다.
- 작성 요령:
  1. 먼저 임시 커밋 해시(혹은 임시 텍스트)를 활용하여 작업한 파일 목록, 수정한 목적, 구체적인 작업 내용을 작성하여 개발자 노트에 추가합니다.
  2. 수정 파일을 스테이징(`git add`)하고, 한글로 명확하게 커밋 메시지를 기재하여 커밋(`git commit`)을 진행합니다.
  3. 커밋 후에 결정된 실제 커밋 해시(앞 7자리)를 확인하여 개발자 노트의 임시 해시를 실제 해시로 교체하고, `git commit --amend --no-edit`로 커밋을 최종 정돈합니다.
  4. 최종 커밋 상태에서 원격 저장소로 `git push`를 수행합니다.
- 이 규칙은 커밋 및 푸시가 요구되는 모든 작업에 항상 무조건적으로 적용됩니다.
