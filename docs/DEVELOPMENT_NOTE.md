# ⚓ DATT 플랫폼 히스토리 백과 (v1.0.0 & v2.0.0 개발자 노트)

## 📁 1. DATT v1.0.0 (`legacy` 브랜치) 개발 히스토리

DATT v1.0.0은 **정박 라이프스타일 플랫폼**의 핵심 가설을 실증하기 위해 구축된 첫 번째 빌드입니다. 주로 기반 데이터 수집 인프라 구축, 핵심 닻(Anchor) 추천 로직 구현, 최소 기능 제품(MVP)의 단일 실행 환경 패키징을 주요 마일스톤으로 삼아 진행되었습니다.

### 🏷️ v1.0.0 주요 피처 및 마일스톤
1.  **초기 인프라 및 DB 도커 세팅**: Postgres와 PostGIS 라이브러리를 바인딩하여 핫플레이스들의 위치 정보(위경도 및 반경 분석)를 연산할 수 있는 기초 공간 연산 데이터베이스 세팅을 정의했습니다.
2.  **소상공인 상권 정보 DB 적재 배치**: 공공데이터 포털에서 수집한 대규모 소상공인 상권 정보를 정제하여 로컬 DB에 안정적으로 마이그레이션 및 적재하는 스프링 배치 모듈을 개발했습니다.
3.  **대표 웹 플랫폼 크롤링 엔진 연동**: 데이터의 신뢰성을 높이고 다양한 장소 메타데이터를 통합 수집하기 위해, 외부 웹 플랫폼의 장소 정보 크롤링 시스템을 모듈 형태로 내장하고 백엔드 엔드포인트와 연결했습니다.
4.  **MVP 프론트엔드 및 닻내리기 기본 기능 개발**:
    *   사용자가 특정 시/도, 시/군/구를 골라 닻을 내리는 UI 및 추천 지역을 지도 기반으로 조회하는 MVP 프론트 기능을 완료했습니다.
    *   생성된 코스를 닻 리포트로 시각화하는 핵심 비즈니스 로직을 구축했습니다.

## 🔍 3. 핵심 챌린지 해결 사례 (Troubleshooting)

### 1) 실서버 배포 시 데이터 유실 버그 극대화 해결
*   **이슈**: CI/CD 배포가 성공할 때마다 데이터베이스 데이터가 유실(초기화)되거나 사용자가 업로드했던 리뷰 이미지들이 모두 사라지는 심각한 부작용 발생.
*   **원인**:
    *   배포 자동화 스크립트(`deploy.yml`)가 로컬 워크스페이스 디렉토리를 통째로 압축(`tar`)하여 실서버로 보낸 후, 기존 디렉토리를 덮어씌워 덮어쓴 시점 이후에 실서버에서 새로 적재된 SQLite/PostgreSQL 데이터베이스 파일 및 업로드 이미지(`uploads`) 디렉토리가 통째로 날아갔음.
    *   또한, Docker Volume 마운트 시 호스트의 절대경로가 아닌 개발용 상대 경로가 바인딩되어 데이터의 영구적인 보존이 격리되지 않았음.
*   **해결책**:
    *   tar 압축 대상 목록에서 `postgres_data`, `uploads`, `nginx/certbot` 등 가변 데이터 폴더를 완전히 영구 배제(`exclude`) 처리.
    *   `docker-compose.yml` 에서 PostgreSQL 볼륨을 도커 관리 독립 볼륨(`datt-db-data`)으로 이관하고 마운트 디렉토리를 `/var/lib/postgresql`에 직접 맵핑하여 격리 보존을 보장함.

### 2) Spring Security Basic Auth 대화상자 인터셉트
*   **이슈**: 프론트엔드가 비인가 요청을 보낼 때 401 Unauthorized 헤더를 수신하는 순간 브라우저 고유의 기본 인증 팝업이 사용자 화면에 난입함.
*   **원인**: 백엔드 스프링 시큐리티 필터 체인에서 기본 인증방식(`.httpBasic(Customizer.withDefaults())`)이 켜져 있어, 인증 실패 시 HTTP 응답 헤더에 `WWW-Authenticate: Basic realm="..."`을 실어 보내 해당 팝업이 강제 기동된 것이 원인이었음.
*   **해결책**: 백엔드 스프링 `SecurityConfig.java`의 설정을 `.httpBasic(AbstractHttpConfigurer::disable)`로 수정하여 브라우저 강제 인증 창을 끄고, 대신 프론트엔드가 Next.js 레이어에서 401 코드를 잡아내 자체 로그인 유도 모달(`LoginGuideModal`)을 부드럽게 띄우는 형태로 전환 설계 완료.
